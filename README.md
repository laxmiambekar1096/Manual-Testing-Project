📌 Manual Testing Project – BrowserStack
📖 Project Overview

This project demonstrates a complete Manual Testing process performed on a web application using structured testing techniques.

It includes Test Scenarios, Test Cases, and Defect Reports, showcasing real-time QA workflow and documentation practices.

🎯 Objective
To validate application functionality through manual testing
To design and execute test scenarios and test cases
To identify, document, and track defects
To demonstrate knowledge of Software Testing Life Cycle (STLC)
📂 Project Files
📊 1. Test Scenarios
Contains high-level scenarios (TS_ID)
Covers different functionalities of the application
Each scenario is mapped to multiple test cases
🧪 2. Test Cases
Detailed test cases (TC_ID)
Includes:
Test steps
Expected results
Actual results (if executed)
Status (Pass/Fail)
Remarks
🐞 3. Defect Report
Defects identified during testing
Includes:
Defect ID (e.g., D_01)
Date reported
Description of issue
Reported by (Laxmi Ambekar)
Status tracking
🔍 Testing Types Performed
Functional Testing
Smoke Testing
Sanity Testing
Regression Testing
🛠️ Tools Used
Microsoft Excel (for test documentation)
GitHub (version control and project sharing)
🔄 Testing Process Followed
Requirement understanding
Test scenario creation
Test case design
Test execution
Defect logging
Reporting
⭐ Key Highlights
Real-time structured QA documentation
Clear mapping between Test Scenarios → Test Cases
Proper Defect Tracking System
Beginner-friendly and interview-ready project
